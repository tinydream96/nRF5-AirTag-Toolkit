## 2025年7月29日 更新：Web UI 功能

### ✨ 新增功能：Web 控制台

为了进一步提升用户体验，本次更新引入了一个基于Web的图形化操作界面。用户现在可以通过浏览器来执行大部分常用的开发任务，无需直接操作命令行。

- **访问方式**:
  1.  进入 `web` 目录: `cd web`
  2.  安装必要的Python库: `pip install Flask`
  3.  启动本地服务器: `python server.py`
  4.  在浏览器中打开: `http://127.0.0.1:5000`

### 🚀 Web 控制台核心功能

1.  **状态总览**:
    *   **环境状态**: 实时检测核心工具链是否完整。
    *   **设备状态**: 自动检测J-Link/ST-Link调试器和目标芯片的连接情况。

2.  **一键环境操作**:
    *   **环境检查**: 图形化展示 `one_click_verify.sh` 的检查结果。
    *   **一键安装**: 在网页上触发 `one_click_install.sh` 脚本，并实时显示安装日志。

3.  **密钥管理**:
    *   **生成密钥**: 输入设备名和数量，即可调用 `generate_device_keys.sh` 生成新的密钥文件。
    *   **查看密钥**: 列出 `config` 目录中所有已生成的设备密钥。

4.  **设备连接与恢复**:
    *   **检查连接**: 确认调试器和芯片的连接状态。
    *   **芯片恢复**: 当芯片被锁定时，可通过网页调用 `recover_protected_chip.sh` 进行恢复。

5.  **固件刷写**:
    *   **自动刷写**: 实现 `auto_flash.sh` 的全部功能，自动处理密钥、恢复和刷写流程。
    *   **手动刷写**: 用户可以从已有的设备密钥列表中选择一个，然后调用 `compile_and_flash_device.sh` 进行刷写。

6.  **重要脚本说明**:
    *   在网页上集成了对核心脚本（如 `auto_flash.sh`, `one_click_verify.sh` 等）的详细说明，方便用户随时查阅。

### 🛠️ 技术实现

- **前端**: 使用 `HTML`, `CSS` 和原生 `JavaScript` 构建，无任何前端框架依赖，轻量且高效。
- **后端**: 一个轻量级的 `Flask` (Python) 服务器 (`web/server.py`)，作为前端和本地Shell脚本之间的桥梁。
- **实时通信**: 使用 `Server-Sent Events (SSE)` 技术，将脚本执行的日志实时推送到前端，提供了流畅的操作体验。

### 📁 新增文件

- `web/index.html`: Web界面的主文件。
- `web/styles.css`: 界面的样式表。
- `web/script.js`: 前端交互逻辑。
- `web/server.py`: 后端Flask服务器。
